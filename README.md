# Windows Terminal Color Scheme

## Installation

### Using Script (if you trust Copilot)

NOTE: This script is generated by Copilot. ~~Powershell's grammar is fxxking too wierd to me~~ . Although tested and works on my computer, please use it at your own risk.

Clone this repository, then run the PowerShell script in repository root:

```powershell
pwsh ./install.ps1
```

The script is interactive and supports:

1. Install repository themes
2. Uninstall repository themes
3. Set dark mode scheme (select from built-in + user + repository schemes)
4. Set light mode scheme (select from built-in + user + repository schemes)

When uninstalling, if configured `colorScheme` references one of this repository's schemes, it will be reset to `Campbell`.

According to [official document](https://learn.microsoft.com/zh-cn/windows/terminal/install#settings-json-file):

> Windows Terminal settings path (`settings.json`) is usually one of:
>
> - Stable: `%LOCALAPPDATA%\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json`
> - Preview: `%LOCALAPPDATA%\Packages\Microsoft.WindowsTerminalPreview_8wekyb3d8bbwe\LocalState\settings.json`
> - Unpackaged (Scoop/Chocolatey): `%LOCALAPPDATA%\Microsoft\Windows Terminal\settings.json`

The script auto-detects these locations and lets you choose when multiple files exist.

### Using Manual Method

Alternatively, you can copy the schemes defination and paste to the settings file (path above, or click settings page (`Ctrl + ,`) > "Open JSON file").

```json
{
    "schemes": [
        // Paste schemes here
    ]
}
```

## Modification for Windows Terminal

Windows Terminal seem to use `white` color instead of `foreground` color to render command parameters, causing it hard to read in some light themes. Not sure if this is a bug on my computer or a design choice.

Anyway, I'm using `black` colors in the original color schemes as `white` colors to make them work better in Windows Terminal.
